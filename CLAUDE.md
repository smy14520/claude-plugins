# CLAUDE.md

Claude Code plugin 仓库。本仓库开发插件，提供栈无关的开发机制；使用插件的项目定义自己的质量标准。。协作时默认使用中文，优先保持流程清晰、状态确定、规则少而有力。
本仓库是 Claude Code 插件框架本体，用于被其他项目依赖。
## 总原则

- **少即是多**：少做不必要的枚举、禁令和解释；给出清晰方向，并给模型留下合理判断空间。
- **规则分层**：CLAUDE.md 只放项目级最高原则；rules 沉淀可复用设计原则；具体阶段步骤放进对应 `SKILL.md`。
- **skill 负责流程，arbor helper 负责机械动作，hook 只守底线**。
- 不要把可由代码、测试或 git 当前状态确认的信息写成长期记忆或重复文档；需要时直接读当前文件。
- 面向用户的 workflow 输出默认使用中文；代码、命令、schema 字段名保持项目既有风格。
- 机制在插件，标准在项目

## 核心原则：机制在插件，标准在项目

**插件提供开发机制**：交付承诺、验证手段、覆盖校验、通过门槛、迭代循环、工具链。

**项目定义自己的标准**：UI 规范、架构原则、设计语言、质量基线、参考范例。写在项目 `CLAUDE.md` / `DESIGN.md` / `.claude/rules/`，harness 自动加载。
