# S-005 管道 CLI

## 交付面

实现 `csv.js` CLI 入口，支持链式操作：

- 命令行格式：
  - `node csv.js --in data.csv select --cols name,amount --filter "amount>100" --sort "amount:desc"`
  - `node csv.js --in data.csv aggregate --group-by category --metric sum --value amount`
  
- 支持操作：select、filter、sort、aggregate（后续可链式组合）

- 行为：
  - 读文件 → 多步操作 → 输出 CSV
  - 文件不存在 → stderr + exit 1
  - 未知操作/列 → stderr + exit 1
  - 空结果 → 打印表头或空、exit 0
  - 合法操作 → stdout 输出 CSV、exit 0

## 验收

- **AC-1**: Given 合法 CSV 文件，When `node csv.js --in data.csv select --cols name,amount --filter "amount>100" --sort "amount:desc"`，Then 输出过滤+排序后的 CSV，exit 0。
  - 失败路径：输出错误或 exit 1。
- **AC-2**: Given 合法 CSV 文件，When `node csv.js --in data.csv aggregate --group-by category --metric sum --value amount`，Then 输出分组聚合后的 CSV，exit 0。
  - 失败路径：聚合错误或 exit 1。
- **AC-3**: Given 输入文件不存在，When 执行任意操作，Then 输出到 stderr 且 exit 1。
  - 失败路径：exit 0 或无 stderr。
- **AC-4**: Given 未知操作或未知列，When 执行，Then 输出到 stderr 且 exit 1。
  - 失败路径：exit 0 或静默忽略。
- **AC-5**: Given 输入合法但结果为空，When 执行，Then 打印表头或空、exit 0。
  - 失败路径：exit 1 或崩溃。

- [assert] cli-aggregate (AC-2): `node --test` 覆盖 aggregate——创建临时 CSV 文件，执行 `node csv.js --in data.csv aggregate --group-by category --metric sum --value amount`，断言 stdout 输出分组聚合后的 CSV 且 exit code 为 0。
- [assert] cli-file-not-found (AC-3): `node --test` 覆盖文件不存在——执行 `node csv.js --in nonexistent.csv select --cols name`，断言 stderr 有输出且 exit code 为 1。
- [assert] cli-unknown-op (AC-4): `node --test` 覆盖未知操作/列——执行未知操作和未知列，断言 stderr 有输出且 exit code 为 1。
