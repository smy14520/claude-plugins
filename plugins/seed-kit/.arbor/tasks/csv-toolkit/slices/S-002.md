# S-002 select + filter

## 交付面

实现 `selectColumns(rows, cols)` 和 `filterRows(rows, conditions)` 两个纯函数：

- `selectColumns(rows: object[], cols: string[]): object[]` — 选指定列：
  - 每行只保留 cols 中的列
  - 未知列抛错（标注未知列名）
  - 空 rows 返回空

- `filterRows(rows: object[], conditions: string[]): object[]` — 条件过滤：
  - 条件格式：`col=value` / `col!=value` / `col>value` / `col<value`
  - 数字列数值比较（检测列值是否全为数字，是则用数值比较）
  - 文本列字典比较
  - 多条件 AND（同时满足才返回）
  - 空 conditions 返回全部
  - 空 rows 返回空

## 验收

- **AC-1**: Given rows，When `selectColumns(rows, ['name','amount'])`，Then 每行只保留这两列。
  - 失败路径：保留其他列或丢失指定列。
- **AC-2**: Given cols 含未知列，When `selectColumns`，Then 抛错（标注未知列名）。
  - 失败路径：静默忽略未知列。
- **AC-3**: Given 数字列 amount，When `filterRows(rows, ['amount>100'])`，Then 数值比较（'99' 不匹配 '>100'，而非字典序）；多条件 AND 组合（同时满足才返回）。
  - 失败路径：字典序比较导致 '99' > '100' 匹配。
- **AC-4**: Given 文本列，When 用 `=`/`!=`/`>`/`<`，Then 按字典序比较。
  - 失败路径：数值比较文本列。
- **AC-5**: Given 空 conditions，When `filterRows(rows, [])`，Then 返回全部；空 rows 返回空。
  - 失败路径：返回空或崩溃。

## 验证

- [assert] select-normal (AC-1): `node --test` 覆盖 `selectColumns` 正常场景——构造 rows 含 name/amount/other 列，`selectColumns(rows, ['name','amount'])` 断言每行只保留这两列，other 列不存在。
- [assert] select-unknown (AC-2): `node --test` 覆盖 `selectColumns` 未知列——调用 `selectColumns(rows, ['name','unknown'])`，断言抛错且错误信息包含 `unknown`。
- [assert] filter-numeric (AC-3): `node --test` 覆盖数字列数值比较——构造 amount 列含 '99' 和 '100'，`filterRows(rows, ['amount>100'])` 断言 '99' 不匹配、'100' 匹配；多条件 AND 断言同时满足才返回。
- [assert] filter-text (AC-4): `node --test` 覆盖文本列字典比较——构造文本列 name 含 'apple'/'banana'，`filterRows(rows, ['name>apple'])` 断言按字典序比较。
- [assert] filter-empty (AC-5): `node --test` 覆盖空 conditions 返回全部、空 rows 返回空。
