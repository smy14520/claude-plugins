# S-004 join + transform

## 交付面

实现 `join(left, right, opts)` 和 `transform(rows, computes)` 两个纯函数：

- `join(left: object[], right: object[], opts: {leftKey: string, rightKey: string, type: 'inner'|'left'}): object[]` — 连接：
  - type ∈ {inner, left}
  - 按 key 匹配合并
  - inner：无匹配的行被排除
  - left：无匹配的 left 行保留，右侧列补空
  - 重复 key：按匹配数量产生相应行（笛卡尔积）
  - 空输入返回空
  - 未知 key 列抛错

- `transform(rows: object[], computes: object): object[]` — 计算新列：
  - `{newCol: (row) => expr}` 从现有列算新列
  - 每行新增 newCol，值由 computes 函数计算
  - 空输入返回空

## 验收

- **AC-1**: Given left 与 right 有匹配 key，When `join(left, right, {leftKey, rightKey, type:'inner'})`，Then 返回匹配合并的行；无匹配的行被排除。
  - 失败路径：未排除无匹配行或合并错误。
- **AC-2**: Given type='left'，When 某 left 行在 right 无匹配，Then 保留该 left 行、右侧列补空。
  - 失败路径：丢弃无匹配行或补空错误。
- **AC-3**: Given 重复 key，When join，Then 按匹配数量产生相应行（不丢不重）。
  - 失败路径：丢失重复或重复计数错误。
- **AC-4**: Given rows，When `transform(rows, {total: r => r.qty * r.price})`，Then 每行新增 total 列，值由现有列计算。
  - 失败路径：total 列缺失或计算错误。
- **AC-5**: Given 空输入，When join/transform，Then 返回空、不崩。
  - 失败路径：崩溃或返回非空。

## 验证

- [assert] join-inner (AC-1): `node --test` 覆盖 inner join——构造 left 和 right 有匹配和无匹配的行，`join(left, right, {leftKey, rightKey, type:'inner'})` 断言返回匹配合并的行，无匹配行被排除。
- [assert] join-left (AC-2): `node --test` 覆盖 left join——构造 left 行在 right 无匹配，`join(left, right, {type:'left'})` 断言保留该 left 行且右侧列补空（值为 undefined 或空字符串）。
- [assert] join-duplicate (AC-3): `node --test` 覆盖重复 key——构造 left 和 right 都有重复 key，断言产生相应行数的笛卡尔积。
- [assert] transform-compute (AC-4): `node --test` 覆盖 transform——构造 rows 和 computes `{total: r => r.qty * r.price}`，断言每行新增 total 列且值正确。
- [assert] join-transform-empty (AC-5): `node --test` 覆盖空输入——join 和 transform 都传入空数组，断言返回空且不崩。
