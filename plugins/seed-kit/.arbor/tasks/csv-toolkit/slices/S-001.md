# S-001 parse + serialize

## 交付面

实现 `parseCsv(text)` 和 `serializeCsv(rows)` 两个核心纯函数：

- `parseCsv(text: string): object[]` — 解析 CSV 文本为行对象数组，处理：
  - 引号包裹的字段（引号内逗号/换行不拆分）
  - 转义引号（`""` → `"`）
  - BOM 剥离（`﻿`）
  - CRLF/LF 统一行分隔（不残留 `\r`）
  - 空文本返回 `[]`
  - 只有表头返回 `[]`
  - 空行跳过
  - 字段数不一致策略：缺补空字符串、多截断（在函数文档中标注）

- `serializeCsv(rows: object[]): string` — 序列化行对象数组为 CSV 文本：
  - 含逗号/引号/换行的值加引号包裹
  - 内部引号 `"` 转义为 `""`
  - 往返一致：`parseCsv(serializeCsv(rows))` 深度等于原 rows

## 验收

- AC-1: Given 含引号字段的 CSV（值内含逗号/换行/转义引号 `""`），When `parseCsv(text)`，Then 每个引号字段作为整体值正确解析（逗号/换行不拆分、`""`→`"`）。
  - 失败路径：引号内逗号被拆分为多字段、换行被截断、`""` 未转为 `"`。
- AC-2: Given 带 BOM 和 CRLF 行尾的文本，When `parseCsv`，Then BOM 被剥离、CRLF 与 LF 都按行分隔、不残留 `\r`。
  - 失败路径：BOM 残留在第一个字段值、`\r` 残留在字段值末尾。
- AC-3: Given 含逗号/引号/换行的 rows，When `serializeCsv(rows)` 再 `parseCsv`，Then 往返深度等于原 rows。
  - 失败路径：往返后字段数减少、值丢失或扭曲。
- AC-4: Given 空文本，When `parseCsv`，Then 返回 `[]`；Given 只有表头无数据行，Then 返回 `[]`。
  - 失败路径：返回非空数组或崩溃。
- AC-5: Given 字段数不一致的行，When `parseCsv`，Then 按标注策略处理（缺补空/多截断）且不崩；空行被跳过。
  - 失败路径：崩溃、字段错位、空行未跳过。

## 验证

- [assert] parse-comma (AC-1): parse CSV with quoted field containing comma and assert field is not split
- [assert] parse-newline (AC-1): parse CSV with quoted field containing newline and assert field contains newline
- [assert] parse-escape-quote (AC-1): parse CSV with escaped quotes and assert double-quote is unescaped
- [assert] parse-bom (AC-2): parse CSV with BOM and assert first field does not contain BOM
- [assert] parse-crlf (AC-2): parse CSV with CRLF and assert no CR remains
- [assert] roundtrip (AC-3): serialize and parse rows and assert deep equality
- [assert] parse-empty (AC-4): parse empty text and assert empty array
- [assert] parse-inconsistent (AC-5): parse rows with inconsistent field count and assert no crash
