# S-003 sort + aggregate

## 交付面

实现 `sortRows(rows, opts)` 和 `aggregate(rows, opts)` 两个纯函数：

- `sortRows(rows: object[], opts: {by: string, order: 'asc'|'desc'}): object[]` — 排序：
  - 数字列数值排序（检测列值是否全为数字，是则用数值比较）
  - order asc/desc
  - 空值排末尾（不论 asc/desc）
  - 未知列抛错

- `aggregate(rows: object[], opts: {groupBy: string, metric: string, value?: string}): object[]` — 聚合：
  - metric ∈ {sum, avg, count, min, max}
  - 按 groupBy 列分组算 value 列
  - sum/avg/min/max 需要 value 列，value 列含非数字抛错（标注非数字）
  - count 不需要 value
  - 未知 groupBy 或 value 列抛错
  - 空组返回空

## 验收

- **AC-1**: Given 数字列 amount 含 '100'/'20'/'3'，When `sortRows(rows, {by:'amount', order:'desc'})`，Then 按数值降序（100>20>3，非字典序 '3'>'20'>'100'）。
  - 失败路径：字典序排序导致 '3' > '20' > '100'。
- **AC-2**: Given 数字列含空值，When 排序，Then 空值排在末尾（不论 asc/desc）。
  - 失败路径：空值排中间或开头。
- **AC-3**: Given 按 category 分组的数据，When `aggregate(rows, {groupBy:'category', metric:'sum', value:'amount'})`，Then 每组 amount 数值求和正确。
  - 失败路径：求和错误或分组错误。
- **AC-4**: Given 数据，When metric ∈ {avg, min, max} on value 列，Then 对应聚合正确；metric='count' 不需要 value 也正常。
  - 失败路径：avg/min/max 计算错误或 count 需要 value。
- **AC-5**: Given value 列含非数字，When metric ∈ {sum, avg, min, max}，Then 抛错（标注非数字）；count 不受影响。
  - 失败路径：静默忽略非数字或 count 报错。

## 验证

- [assert] sort-numeric (AC-1): `node --test` 覆盖数字列数值排序——构造 amount 列含 '100'/'20'/'3'，`sortRows(rows, {by:'amount', order:'desc'})` 断言顺序为 100>20>3，非字典序。
- [assert] sort-nulls-last (AC-2): `node --test` 覆盖空值排末尾——构造含空值的数字列，asc 和 desc 排序都断言空值在最后。
- [assert] aggregate-sum (AC-3): `node --test` 覆盖 sum 聚合——构造分组数据（category/amount），`aggregate({groupBy:'category', metric:'sum', value:'amount'})` 断言每组求和正确。
- [assert] aggregate-avg-min-max (AC-4): `node --test` 覆盖 avg/min/max/count 聚合——断言 avg/min/max 计算正确，count 不需要 value。
- [assert] aggregate-non-numeric (AC-5): `node --test` 覆盖非数字 value 抛错——构造 value 列含非数字，sum/avg/min/max 断言抛错且错误信息标注非数字，count 断言不报错。
