**以下相邻能力里，哪些应该纳入本次 MVP？**

我的推荐是：**只纳入空态提示和基础可访问性；其余全部 out of scope。**

理由：  
空态直接影响首次使用；基础可访问性影响 checkbox、按钮、输入框能否正常键盘操作和识别。搜索、批量操作、导入导出、快捷键、主题等都不是“快速记录 + 三分组整理”的必要条件。

可选方向：

1. **只纳入空态 + 基础可访问性（推荐）**：空列表有提示；输入框、按钮、勾选、分组切换具备基本 label/键盘可用性。
2. **再加键盘快捷键**：例如 Enter 新增、Esc 取消编辑；提升效率，但会增加交互规则。
3. **再加导入/导出 JSON**：降低本地数据丢失风险，但会扩大数据管理 scope。
4. **再加搜索/批量清理**：适合任务很多时使用，但偏离轻量 MVP。

### AI user

选 **1. 只纳入空态 + 基础可访问性**。

MVP 里只需要把这两类相邻能力纳入：

- **空态提示**
  - 首次打开、没有任何任务时，页面要有清楚提示，比如“还没有任务，先添加一个今天要做的事”。
  - 如果某个分组为空，也可以在该分组里显示轻量提示，比如“今天暂无任务”。
  - 如果开启“隐藏已完成”后某个分组没有可见任务，也要避免看起来像页面坏了，可以提示“没有未完成任务”。

- **基础可访问性**
  - 新增输入框、按钮、完成勾选、编辑、删除、分组切换都要能通过键盘正常操作。
  - 表单控件要有基本可识别的 label 或 aria-label。
  - checkbox 状态要清楚表达已完成 / 未完成。
  - 删除、重置本地数据这类有破坏性的操作，要有明确按钮文案和确认，不要只靠图标。

其余都明确 **out of scope**：

- 不做搜索
- 不做批量选择 / 批量删除 / 批量移动
- 不做导入 / 导出 JSON
- 不做自定义快捷键体系
- 不做主题切换 / 深色模式
- 不做统计、完成历史、归档
- 不做多设备同步或账号能力

业务上我还是希望这个 MVP 聚焦在：**快速记录任务、按今天 / 本周 / 以后整理、可靠保存在本地**。空态和基础可访问性是产品可用性的底线，其他能力可以后续再评估。
