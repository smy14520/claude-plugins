---

检查并更新经验文档的元数据，确保经验库的完整性和准确性。

## 用法

```
/meta-maintain              # 检查所有
/meta-maintain --auto-fix   # 自动修复可修复的问题
```

等价于：`$ARGUMENTS` 为选项参数。

## 检查项

| 检查项 | 说明 | 可自动修复 |
|--------|------|-----------|
| files_exist | 引用文件是否存在 | ❌ |
| updated | 日期是否准确 | ✅ |
| related | 引用文档是否存在 | ✅ |
| content_stale | 内容是否过时 | ❌ |

## 每个检查项如何执行

### 1. files_exist（引用文件存在性检查）

1. 遍历 `.claude/experience/` 下所有经验文档
2. 从 `files` 字段提取文件路径列表
3. 检查每个文件是否存在于项目中
4. 报告不存在的文件路径

### 2. updated（日期准确性检查）

1. 获取文档的 `updated` 字段
2. 对比文档文件的实际修改时间
3. 差异超过 7 天则标记为需要更新

### 3. related（引用文档存在性检查）

1. 从 `related` 字段提取相关文档列表
2. 检查每个文档是否存在于经验库中
3. 报告不存在的相关文档

### 4. content_stale（内容过时检查）

1. 分析文档中的代码示例
2. 对比项目中的实际代码
3. 语义分析判断内容是否需要更新

## 报告格式

```
【经验文档元数据检查报告】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

检查文档数: 15
发现问题: 4

❌ 严重问题 (1):
   OAuth最佳实践.md - 缺失引用文件

⚠️ 警告问题 (3):
   用户认证模块.md - 日期需要更新
   数据库优化.md - 无效相关文档
   缓存策略.md - 内容可能过时

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
使用 --auto-fix 自动修复可修复的问题
```

## 建议执行频率

每周执行一次 `/meta-maintain` 检查，保持经验库的健康状态。
