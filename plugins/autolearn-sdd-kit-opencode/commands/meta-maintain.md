---
description: 检查并更新经验文档的元数据
---

检查并更新经验文档的元数据，确保经验库的完整性和准确性。

## 用法

```
/meta-maintain              # 检查所有
/meta-maintain --auto-fix   # 自动修复可修复的问题
```

## 检查项详情

| 检查项 | 说明 | 可自动修复 |
|--------|------|-----------|
| files_exist | 引用文件是否存在 | ❌ |
| updated | 日期是否准确 | ✅ |
| related | 引用文档是否存在 | ✅ |
| summary_kind | `summary/kind` 是否缺失或无效 | ✅ |
| tags_quality | tags 是否为空、过多或信号过弱 | ❌ |
| index_drift | INDEX 与文档元数据是否不一致 | ✅ |
| content_stale | 内容是否过时 | ❌ |

## 每个检查项如何执行

### 1. files_exist（引用文件存在性检查）

1. 遍历所有经验文档
2. 从 `files` 字段提取文件路径列表
3. 检查每个文件是否存在于项目中
4. 报告不存在的文件路径

### 2. updated（日期准确性检查）

1. 获取文档的 `updated` 字段
2. 对比文档文件的实际修改时间
3. 如果差异超过 7 天，标记为需要更新

### 3. related（引用文档存在性检查）

1. 从 `related` 字段提取相关文档列表
2. 检查每个文档是否存在于经验库中
3. 报告不存在的相关文档

### 4. summary_kind（最小检索元数据检查）

1. 检查文档是否包含 `summary` 与 `kind`
2. 验证 `summary` 是否为一句话短摘要
3. 验证 `kind` 是否属于 `pattern | gotcha | decision | checklist`

### 5. tags_quality（标签质量检查）

1. 检查 `tags` 是否为空
2. 检查标签数量是否明显过多（如超过 5 个）
3. 检查是否存在过于泛化、无法支持检索的标签

### 6. index_drift（索引漂移检查）

1. 读取 `INDEX.md` 中对应文档条目
2. 对比名称、标签、关键文件、类型、更新日期、摘要是否与 frontmatter 一致
3. 报告不一致项

### 7. content_stale（内容过时检查）

1. 分析文档内容中的代码示例
2. 对比项目中的实际代码
3. 检查关键 API 是否有变化
4. 语义分析判断内容是否需要更新

## 维护边界

这是一个纯 checker / maintainer 命令：
- 只检查高价值元数据与索引一致性问题，不做重型内容审查
- 不做闲聊式澄清，不输出与检查无关的后续建议
- 如果命令已经被明确触发，直接进入检查流程，不要停在命令说明或菜单式输出上
- 目标是提升检索质量和控制知识膨胀，而不是把每篇经验文档都重写一遍

## 完整报告示例

```
【经验文档元数据检查报告】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

检查文档数: 15
发现问题: 4

❌ 严重问题 (1):
   OAuth最佳实践.md - 缺失引用文件

⚠️ 警告问题 (3):
   用户认证模块.md - 日期需要更新
   数据库优化.md - 无效相关文档
   缓存策略.md - 内容可能过时

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
使用 --auto-fix 自动修复可修复的问题
```

## 建议执行频率

每周执行一次 `/meta-maintain` 检查，保持经验库的健康状态。
