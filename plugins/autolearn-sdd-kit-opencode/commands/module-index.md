为当前项目生成或更新项目内的模块索引文件。

## 用法

```
/module-index <模块路径>
/module-index <模块路径> --update

示例：
/module-index src/modules/auth
/module-index src/modules/notes --update
```

等价于：`$ARGUMENTS` 包含模块路径和可选 `--update`。

## 路径说明

索引存储在项目目录：`./.claude/modules/`

## 执行要求

1. 从 `$ARGUMENTS` 中解析出：
   - 目标模块路径
   - 是否包含 `--update`
2. 先只围绕**用户给定路径本身**读取代码。默认只读取：
   - 该目录下的文件
   - 直接引用到的本地文件
   - 直接调用它的 1 跳入口文件
3. 禁止为了“补全常见 auth / service / repository 惯例”而搜索当前代码库里并不存在的方法名、接口名或模式。
4. 不要只做目录摘要；必须基于真实代码与调用关系判断模块边界。
5. 如 `./.claude/modules/` 不存在，先创建它。
6. 生成或更新：
   - 全局索引：`./.claude/modules/INDEX.md`
   - 模块主索引：`./.claude/modules/<module-name>-index.md`
   - 如果是重模块，再按需生成：
     - `./.claude/modules/<module-name>/INDEX.md`
     - 最多 4 个子索引文件
7. 如果是 `--update`，优先保留已有高价值信息，只删除过时、重复、机械化内容。
8. 完成后只输出：
   - 模块类型：`light` 或 `heavy`
   - 写入的文件路径
   - 一句下一步建议

## 模块分类规则

### 轻模块（light）
满足以下大多数特征时，判为轻模块：
- 职责单一
- 目录浅
- 没有多个稳定业务子域
- 没有复杂状态流
- 没有明显特殊流/例外分支
- 外部依赖少

**产出**：只生成一个主索引文件。

### 重模块（heavy）
满足以下任意 **2 条以上**，且这些特征都能从**当前模块目录自身 + 一跳调用链**明确观察到，才判为重模块：
- 存在 3 个以上稳定业务子域
- 有明确核心主链路
- 有多角色 / 多状态 / 多分流
- 有明显缓存 / Event / Job / Listener 副作用
- 有多个外部依赖系统
- 存在高频误判点或特殊流

如果只是“方法较多”“有 repository + service 两层”“被别的模块调用较多”，**仍优先判为 light**。

**产出**：主索引 + 子模块索引目录 + 2–4 个子索引。

## 拆分原则

- 优先按**业务子域**拆，不优先按目录拆
- `INDEX.md` 只做查找与定位，不重复正文
- 不要为每个类/方法逐个解释
- 不做三层以上递归索引
- 优先写“AI 不容易从目录树和函数名直接推出”的信息

## 主索引要求

### 轻模块主索引必须回答
- 负责什么
- 目录结构是什么
- 关键入口是什么
- 关键类/服务是什么
- 依赖什么

### 重模块主索引必须回答
- 模块整体职责
- 稳定子域有哪些
- 核心主链路是什么
- 外部入口在哪
- 核心类/服务如何分工
- 依赖哪些模块/系统
- 哪些点最容易误判
- 继续深挖应该看哪个子索引

## 输出格式要求

### 轻模块主索引
文件名：`./.claude/modules/<module-name>-index.md`

必须使用以下结构，不要自由发挥标题：

```markdown
# <module-name> 模块

> 路径: `<module-path>`
> 更新: <YYYY-MM-DD>
> 类型: 轻模块索引

## 职责
...

## 目录结构
```text
...
```

## 关键入口
...

## 关键类/服务
...

## 依赖
...

## 外部入口与调用关系
...

## 模块边界判断
...

## 容易误判的点
...
```

### 重模块主索引
文件名：`./.claude/modules/<module-name>-index.md`

必须使用以下结构：
- 标题：`# <module-name> 模块`
- 元信息：路径 / 更新 / 类型 / 子索引
- 小节：职责 / 子域地图 / 核心主链路 / 模块分层理解 / 关键入口 / 关键状态 / 外部依赖 / 易混淆点 / 推荐继续下钻的子模块索引

### 全局索引
文件名：`./.claude/modules/INDEX.md`

必须使用以下结构：

```markdown
# 模块索引

> 最后更新: <YYYY-MM-DD>
> 模块总数: <数量>

| 模块 | 路径 | 索引文件 | 类型 | 一句话职责 | 子索引 |
|------|------|----------|------|-----------|--------|
```

`./.claude/modules/INDEX.md` 只保留：
- 元信息
- 一张表格：模块、路径、索引文件、类型、一句话职责、子索引数量

不要把主链路、依赖、风险点等正文塞进全局索引。
