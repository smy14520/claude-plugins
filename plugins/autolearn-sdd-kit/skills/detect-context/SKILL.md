# detect-context

按需精准检索项目上下文，让 AI 了解与当前任务相关的信息。

## 触发场景

当用户说以下内容时自动触发：
- "帮我了解项目上下文"
- "检索一下相关经验"
- "看看有什么风险提示"
- "调用上下文侦探"

## 精准检索策略

**核心原则：只加载相关的，不要全部加载。**

### Step 1: 提取关键词

从用户的需求描述中提取关键词：
- 技术关键词（OAuth, React, Redis 等）
- 功能关键词（登录, 支付, 用户管理等）
- 操作关键词（重构, 优化, 迁移等）

### Step 2: 按关键词精准匹配

1. **识别项目**：`project = basename(pwd)`

2. **匹配规则** → `./.claude/rules/risk-rules.md`
   - 仅匹配 trigger 关键词与需求关键词有交集的规则
   - ❌ 不要返回不相关的规则

3. **匹配经验** → `./.claude/experience/INDEX.md`
   - 按 Tag 和模块名与关键词匹配
   - 最多返回 TOP 3 篇
   - ❌ 不要加载全部经验

4. **匹配模块索引** → `./.claude/modules/`
   - 仅加载路径或模块名匹配的索引
   - 最多返回 TOP 3 个
   - ❌ 不要加载全部模块索引

## 输出格式

```
【ContextDetective 报告】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

项目: <项目名>
需求: <需求描述>
关键词: [提取的关键词列表]

⚠️ 风险提示 (N):
  • <风险提示1>
  （无匹配时输出：无相关风险规则）

📄 相关经验 (N/最多3):
  • <经验文档1> — 匹配: <Tag/模块>
  （无匹配时输出：无相关经验）

📦 模块索引 (N/最多3):
  • <模块索引1>
  （无匹配时输出：无相关模块）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
