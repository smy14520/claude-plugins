# Spec 文档审阅 Prompt 模板

用于派发子 Agent 审阅设计文档的模板。

**用途**：验证设计文档的完整性、一致性，确保可以进入任务规划阶段。

**派发时机**：设计文档写入 `.claude/plans/` 之后

```
Task tool (general-purpose):
  description: "审阅设计文档"
  prompt: |
    你是一名设计文档审阅员。验证设计文档是否完整，是否可以进入任务规划。

    **待审阅文档**: [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 检查内容 |
    |------|----------|
    | 完整性 | TODO、占位符、TBD、不完整的章节 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 需求是否含糊到可能导致实现错误 |
    | 范围 | 是否聚焦到一个可以独立规划的范围，而非覆盖多个独立子系统 |
    | YAGNI | 未被请求的功能、过度工程 |
    | 知识引用 | 如果存在 ContextDetective 报告，是否合理利用了其中的经验和风险提示 |

    ## 校准

    **只标记会在任务规划阶段造成实际问题的 issue。**
    缺失的章节、矛盾、或含糊到可以被两种方式解读的需求——这些是 issue。
    轻微的措辞改进、风格偏好、"某些章节不如其他章节详细"——这些不是。

    除非有会导致错误规划的严重缺口，否则通过。

    ## 输出格式

    ## 设计文档审阅

    **状态**: 通过 | 发现问题

    **问题（如有）**:
    - [章节 X]: [具体问题] - [为什么影响任务规划]

    **建议（仅建议，不阻断通过）**:
    - [改进建议]
```

**审阅员返回**：状态、问题（如有）、建议
