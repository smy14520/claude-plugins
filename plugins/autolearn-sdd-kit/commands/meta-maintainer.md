# /meta-maintain

检查并更新经验文档的元数据，确保经验库的完整性和准确性。

## 用法

```bash
/meta-maintain              # 检查所有
/meta-maintain --auto-fix   # 自动修复可修复的问题
/meta-check                 # 仅检查不修复
```

## 检查项详情

| 检查项 | 说明 | 执行方式 | 可自动修复 |
|--------|------|----------|-----------|
| files_exist | 引用文件是否存在 | 检查相关文件路径 | ❌ |
| updated | 日期是否准确 | 对比文件修改时间 | ✅ |
| related | 引用文档是否存在 | 验证 related 文档存在 | ✅ |
| content_stale | 内容是否过时 | 语义分析需人工判断 | ❌ |

## 每个检查项如何执行

### 1. files_exist（引用文件存在性检查）

**执行流程**：
1. 遍历所有经验文档
2. 从 `files` 字段提取文件路径列表
3. 检查每个文件是否存在于项目中
4. 报告不存在的文件路径

**输出示例**：
```
❌ OAuth最佳实践.md
   缺失文件: app/Service/OAuthService.php (已重命名)
   缺失文件: config/oauth.php (已删除)
```

**修复方式**：手动更新文件路径或删除无效引用

### 2. updated（日期准确性检查）

**执行流程**：
1. 获取文档的 `updated` 字段
2. 对比文档文件的实际修改时间
3. 如果差异超过 7 天，标记为需要更新

**输出示例**：
```
⚠️ OAuth最佳实践.md
   记录日期: 2024-01-15
   实际修改: 2024-02-01
   建议更新日期
```

**自动修复**：使用 `--auto-fix` 时自动更新为当前日期

### 3. related（引用文档存在性检查）

**执行流程**：
1. 从 `related` 字段提取相关文档列表
2. 检查每个文档是否存在于经验库中
3. 报告不存在的相关文档

**输出示例**：
```
⚠️ OAuth最佳实践.md
   无效引用: GitHub登录旧版.md (已删除)
```

**自动修复**：使用 `--auto-fix` 时自动删除无效引用

### 4. content_stale（内容过时检查）

**执行流程**：
1. 分析文档内容中的代码示例
2. 对比项目中的实际代码
3. 检查关键 API 是否有变化
4. 语义分析判断内容是否需要更新

**输出示例**：
```
⚠️ OAuth最佳实践.md
   可能过时: 代码示例使用旧版 API
   建议: 审查并更新示例代码
```

**修复方式**：需要人工审查和更新

## 建议执行频率

**每周执行一次** `/meta-maintain` 检查，保持经验库的健康状态。
